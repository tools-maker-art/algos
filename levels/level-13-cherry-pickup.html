🍒 Two Friends, One Orchard

Cherry Pickup

World B Grid DP

🎮 Objective

Two robots walk a grid simultaneously, picking cherries. Maximise the total they collect.

🧠 Problem (in tiny words)

Grid with cherries. Two paths from (0,0) to (m-1,n-1), each Right/Down. Cells visited by both count once.

🎭 Story

Two friends start at the same gate. They each take a path. They’re not allowed to take the same cherry twice.

😵 Brute force (the painful way)

Try all pairs of paths — quadratic explosion.

🌱 From brute → recursion

Move both simultaneously by step count. State = (step, r1, r2). c1 = step-r1, c2 = step-r2. Pick max over the four (down/right × down/right) parents.

🔁 Recursion

dp(step, r1, r2) = grid[r1][c1] + (r1==r2 ? 0 : grid[r2][c2]) + max(four parents).

# pseudo
dp(step, r1, r2) = grid[r1][c1] + (r1==r2 ? 0 : grid[r2][c2]) + max(four parents).

🧩 Memoization

3D cache.

🧱 Tabulation

3D fill by step.

🚀 Space optimisation

Two 2D layers (current step / previous).

📊 Complexity

O(n³) time, O(n²) space (with optimisation).

💡 Key takeaway

When two agents move together, model state as (step, position1, position2).

🎯 Mini challenge

What if a cell is a wall? (Make its value -∞ and skip those states.)

🎮 Game UI idea

Two coloured paths overlapping on a grid. Cherries glow when collected.
