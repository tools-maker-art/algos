Level 01

🐣 The Bunny Family Tree

Fibonacci Number

World A Basics

🎮 Objective

Find the N-th number in the famous Fibonacci sequence — without falling asleep counting.

🧠 Problem (in tiny words)

Fibonacci is a list where each number is the sum of the two before it: 0, 1, 1, 2, 3, 5, 8, 13… We want the N-th one.

🎭 Story

Two bunnies appear. Every month, every pair makes a new pair. How many pairs after N months? That count is the Fibonacci sequence. Cute and sneaky.

😵 Brute force (the painful way)

Write fib(n) = fib(n-1) + fib(n-2). Looks tiny — but fib(40) recomputes the same little numbers billions of times. Your laptop sweats.

🌱 From brute → recursion

We notice the same fib(k) is asked over and over. So… write the answer down the first time, then reuse it. That’s the leap from recursion to memoization.

🔁 Recursion

fib(n) = fib(n-1) + fib(n-2), base: fib(0)=0, fib(1)=1.

# pseudo
fib(n) = fib(n-1) + fib(n-2), base: fib(0)=0, fib(1)=1.

🧩 Memoization

Keep an array memo[]. Before computing fib(n), check memo[n]. If known, return it. Else compute, store, return.

🧱 Tabulation

Build dp[0..n] left to right: dp[0]=0, dp[1]=1, dp[i]=dp[i-1]+dp[i-2].

🚀 Space optimisation

Only the last two values matter. Keep two variables: prev2, prev1. O(1) memory.

📊 Complexity

Brute recursion: O(2ⁿ) — explosive. Memo / Tab / Space-optimized: O(n) time, O(1) memory after optimisation.

💡 Key takeaway

If you ever recompute the same thing twice, you’re ready for DP.

🎯 Mini challenge

Find fib(20) using only two variables. (Answer: 6765.)

🎮 Game UI idea

Recursion tree: tap a node to mark it ‘cached’. Watch how memoization prunes giant duplicate subtrees.
