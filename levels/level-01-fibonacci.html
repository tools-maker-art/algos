// Try every choice. Sweet, simple — and slow.
int ways(int n) {
    if (n == 0) return 1;   // already at the top → 1 way
    if (n == 1) return 1;   // only one tiny hop possible

    // Try hopping 1 stair  OR  hopping 2 stairs
    return ways(n - 1) + ways(n - 2);
}

// Problem: ways(3) gets called many times.
// For n = 30 this is ~1.6 million calls. Yikes.

🎒 Backpack (answers we already found)

what does “tiny problem” mean?

The big problem is “5 stairs, how many ways?”

To answer it, we have to first know:

🪜 4 = ?a smaller version of the same puzzle
🪜 3 = ?an even smaller version
🪜 2 = ?tinier still
🪜 1 = 1easy! just one way
🪜 0 = 1already at the top, count this way

Each of those is a tiny problem. Memoization just means: write each tiny answer down once and reuse it.

👀 see the Java code (step 2: + memory)
JAVA   recursion + memoization ⬆ each tiny problem solved once 
// New: a "backpack" array to remember answers.
int[] memo;

int ways(int n) {
    if (n == 0 || n == 1) return 1;

    if (memo[n] != -1) return memo[n];   // ✨ already in backpack!

    memo[n] = ways(n - 1) + ways(n - 2);
    return memo[n];
}

int solve(int n) {
    memo = new int[n + 1];
    Arrays.fill(memo, -1);   // -1 means "not solved yet"
    return ways(n);
}

// Now ways(3) is computed ONCE and reused.
// 1.6 million calls → about 30 calls for n = 30.

🧱 Tabulation: build a small table from scratch

Forget recursion. Make a row of boxes, one per stair count. Fill them left → right. Each new box = sum of the last two boxes.

what is “tabulation”?

It’s like filling a tiny scoreboard:

0 stairs → 1 1 stair → 1 2 stairs → 1+1 = 2 3 stairs → 1+2 = 3 4 stairs → 2+3 = 5 5 stairs → 3+5 = 8

We never recurse. We just walk the row left to right, adding the last two boxes. Done!

👀 see the Java code (step 3: no recursion at all)
JAVA   bottom-up tabulation ⬆ no recursion · just a loop 
int ways(int n) {
    int[] dp = new int[n + 1];

    dp[0] = 1;   // base: 0 stairs → 1 way
    dp[1] = 1;   // base: 1 stair  → 1 way

    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];   // last two boxes
    }

    return dp[n];
}

// We literally fill the scoreboard left → right.
// No call stack, no risk of recursion blowing up.

what is the “space saver”?

Imagine remembering a long list vs. just remembering the last two numbers in your head — much easier!

prevthe answer two stairs back
currthe answer one stair back
nextprev + curr = the new answer

Then we slide: prev becomes curr, curr becomes next, repeat. Same answer as the table — but we never need more than 2 boxes of memory.

👀 see the Java code (step 4: only 2 ints!)
JAVA   space-optimised DP ⬆ only 2 ints of memory 
int ways(int n) {
    if (n == 0 || n == 1) return 1;

    int prev = 1;   // answer for (i - 2)
    int curr = 1;   // answer for (i - 1)

    for (int i = 2; i <= n; i++) {
        int next = prev + curr;   // new answer
        prev = curr;                // slide forward
        curr = next;
    }

    return curr;
}

// Same answer. Same speed. Just 2 ints of memory.
